#+TITLE: Dashboard Feature - Quick Reference
#+DATE: 2026-02-11

* Overview

Overview and monitoring feature. Displays stats, alerts, and recent activity for the client's tenders.

** useRelevantTenders Hook Parameters

The hook passes the following fixed parameters to get_tenders_paginated to ensure only active, not-yet-closed tenders appear:

| Parameter | Value | Purpose |
|-----------|-------|---------|
| p_is_relevant | true | Filter to relevant tenders only |
| p_status | 'active' | Explicitly filter by active status |
| p_min_days_until_close | 0 | Ensure closing_date >= NOW() (not yet closed) |
| p_sort_by | 'score' | Sort by relevance score (best first) |
| p_sort_desc | true | Descending order (highest scores first) |
| p_limit | limit parameter (default 12) | Control total tenders fetched for display |
| p_offset | 0 | Start from first result |

All other parameters (province_id, department_id, category_id, closing_from, closing_to, has_compulsory_briefing, search_text) are set to null to retrieve all relevant tenders without additional filtering.

Hook interface parameters:
| Parameter | Type | Default | Purpose |
|-----------|------|---------|---------|
| limit | number (optional) | 12 | Maximum tenders to fetch |

* Component Features

** TenderCarousel

| Feature | Implementation |
|---------|----------------|
| Data source | useRelevantTenders hook (default limit: 12) |
| Grouping | groupTendersByRecency() splits tenders into "New This Week" (last 7 days) and "Earlier", each sorted most recent first |
| Carousel mode | Renders two independent CarouselSection components, one per group (hidden if empty) |
| Grid mode | Shows both groups in responsive grid (1/2/3 columns), with divider between sections if both exist |
| Auto-scroll | Per-section 5000ms interval, configurable via autoScrollInterval prop |
| Pause behavior | Per-section pause on mouse hover, resume on mouse leave |
| Responsive | 1 card (mobile), 2 cards (tablet), 3 cards (desktop) |
| Navigation | Prev/Next buttons (w-8 h-8) and dot indicators per section |
| Card navigation | Click card to navigate to /tenders/:tender_pk |
| View toggle | Button to switch between carousel and grid, positioned top-right of card |
| Hover effect | Carousel container allows overflow-y-visible to support card hover lift |

** TenderCarouselCard

| Feature | Implementation |
|---------|----------------|
| Hover effect | 2px primary border with transition-colors on card container |
| Score badge | Dynamic styling via getScoreBadgeStyle() based on percentage |
| Score tiers | >=80% (success green), >=60% (info blue), else (warning amber) |
| Badge styling | Returns combined classes: text-{tier} border-{tier} bg-{tier}/10 |
| Urgency badge | Days until close colored red (<7d), amber (7-14d), primary (14+d) |

** ScoreDistributionChart

| Feature | Implementation |
|---------|----------------|
| Chart type | Recharts donut chart (PieChart with innerRadius) |
| Data source | useDashboardStats hook |
| Colors | Excellent Fit (dark gold hsl(32,80%,40%)), Good Fit (medium gold hsl(45,93%,50%)), Worth Reviewing (light gold hsl(48,85%,65%)) |
| Labels | Percentage text with dark fill (hsl(240,10%,10%)) and white stroke, shown if >5% |
| Legend | Custom legend with colored squares and counts below chart |

** DepartmentTreemap

| Feature | Implementation |
|---------|----------------|
| Chart type | Recharts BarChart (horizontal layout) |
| Data source | useFilterOptions hook (departments with counts) |
| Display limit | Top 10 departments by count, then "Other" group |
| Colors | 10-color gold/amber palette, cycles through bars |
| Labels | Department name (Y-axis), count at end of bar |
| Height | Fixed 320px, scrollable if needed |

** StatCard Props

| Prop | Type | Purpose |
|------|------|---------|
| label | string | Metric label text |
| value | number or string | Metric value in serif font |
| featured | boolean (optional) | Adds gradient border and bg if true (default: false) |

* Dashboard Page Layout

DashboardPage.tsx displays components in this order:
1. StatsGrid + AlertsRow (flex row on large screens)
2. TenderCarousel (auto-scrolling carousel or expanded grid)
3. ScoreDistributionChart + DepartmentTreemap (side-by-side on desktop)
4. RecentChanges (activity status card)

Note: ClosingTimelineChart component exists but is not currently integrated in DashboardPage.

* CarouselSection Component

Internal component within TenderCarousel that renders a single section (group) with independent carousel behavior.

| Prop | Type | Purpose |
|------|------|---------|
| tenders | TenderListItem[] | Array of tenders to display in this section |
| title | string | Section header text ("New This Week" or "Earlier") |
| autoScrollInterval | number | Interval in ms for auto-scroll (passed from parent) |

| Feature | Implementation |
|---------|----------------|
| Auto-scroll | Triggers goNext() every autoScrollInterval ms if not paused |
| Pause behavior | Paused on mouse enter, resumed on mouse leave |
| Navigation visibility | Prev/Next buttons and dots shown only if tenders.length > visibleCount |
| Responsive layout | Updates visibleCount on window resize: 1 (mobile), 2 (tablet), 3 (desktop) |
| Keyboard-friendly | Dot indicators clickable to jump to slide, nav buttons with aria-labels |
| Reset behavior | Index resets to 0 when tenders prop changes |
