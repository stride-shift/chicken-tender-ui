* Semantic UI Components

** PixelBox

Container component (=ui/PixelBox.tsx:8=) with semantic styling:

- Border: border-border, rounded-lg
- Background: bg-card
- Text: text-card-foreground
- Shadow: shadow-sm
- No longer accepts color/bgColor props

** LEDNumber

Display component (=ui/LEDNumber.tsx:7=) for numeric values:

- font-semibold tabular-nums
- Three sizes: small (text-xl), medium (text-3xl), large (text-5xl)
- Color prop unused (prefixed _color)
- Semantic foreground color

** HealthBar

Progress component (=ui/HealthBar.tsx:10=) with single bar:

- Track background: bg-muted-foreground/20 for better visibility
- Progress bar: bg-primary with smooth 500ms transition
- Current/max percentage calculation
- Optional numeric label: current/max

** ArcadeButton

Button component (=ui/ArcadeButton.tsx:41=) with semantic variants:

- Five variants: primary, secondary, success, warning, danger
- Three sizes: sm (h-9), md (h-10), lg (h-11)
- Semantic color classes: bg-primary, text-primary-foreground, etc.
- Icon animation support: wiggle, bounce, spin, none
- Disabled state: opacity-50 cursor-not-allowed

** PixelStrip

Divider component (=ui/PixelStrip.tsx:6=):

- Returns simple h-[1px] w-full bg-border
- Color/segments props unused

** Scanlines

Empty stub (=ui/Scanlines.tsx:5=):

- Returns null (no-op)
- CRT effect removed

** Logo

Branding component (=ui/Logo.tsx:47=) with StrideShift and TenderRender text:

- Icon: circular border with dot in black (#17171c)
- Text: STRIDESHIFT in black (#17171c), GLOBAL in dark gray (#555568)
- TENDER in black (#17171c), RENDER in medium-dark gray (#6b6b78)
- Four size variants: sm, md, lg, xl with proportional scaling
- showParentBrand prop controls display of parent branding

* Layout Structure

AppShell (=layout/AppShell.tsx:10=) orchestrates layout:

- Renders Header at top with sidebar toggle callback
- Renders Sidebar with collapse state
- Wraps children with margin based on sidebar state
- Sidebar collapsed: ml-16 (64px), expanded: ml-56 (224px)

Header (=layout/Header.tsx:10=) provides:

- sticky top-0 bg-card border-b border-border shadow-sm
- Burger menu button with hover:bg-muted
- Logo component at size "lg"
- Breadcrumb: "/" + current page name (Dashboard/Tenders/Rubric)
- Profile pill button: avatar (bg-primary) + name + client name + dropdown
- Profile dropdown contains:
  - User display name and admin status in text-accent (StrideShift only)
  - Client selector list with active highlight bg-accent/10 text-accent, dot and checkmark in text-accent
  - Sign Out button with bg-destructive
- Dropdown closes on click-outside (useRef + useEffect)
- Uses useAuth for clients, switching, profile, logout

Sidebar (=layout/Sidebar.tsx:56=) manages navigation:

- Fixed position: bg-card border-r border-border
- Transitions width on collapse: 64px (collapsed) / 224px (expanded)
- Navigation items: Dashboard (/), Tenders (/tenders), Rubric (/rubric)
- Active route: bg-primary/10 text-primary font-medium
- Inactive hover: bg-muted text-foreground

* Error Handling

** ErrorBoundary

Class component (=shared/ErrorBoundary.tsx:49=) catching React errors:

- Wraps component trees to prevent full app crashes
- Uses getDerivedStateFromError to capture error state
- Logs errors via componentDidCatch
- Renders custom fallback or default error UI
- Default fallback includes reload button

** ErrorState

Functional component (=shared/ErrorState.tsx:7=) for explicit error display:

- Default title "Error", message "Something went wrong"
- Simple centered layout with text-center
- Optional retry button: bg-destructive text-destructive-foreground
- Message in destructive color for error title
- No longer uses PixelBox wrapping

** LoadingState

Functional component (=shared/LoadingState.tsx:7=) for loading display:

- Animated dot pulse: 3 dots with staggered 0.15s delay
- Each dot: w-2 h-2 bg-primary rounded-full animate-pulse
- Optional message text in text-muted-foreground

** Toast Notifications

Toast system (=ui/Toast.tsx:39=, =ui/Toaster.tsx:4=) for transient messages:

- Three types: success (✓), error (✕), info (ℹ)
- Type colors: success, destructive, info via semantic classes
- Left border indicator: border-l-4 border-l-{type}
- Icon + label + message + close button layout
- Managed via ToastContext, fixed bottom-right z-50
